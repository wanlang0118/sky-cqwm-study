🛠️ 一、技术架构全景图
项目采用经典的前后端分离与分布式微服务雏形架构，技术选型覆盖了企业级应用的核心环节。

技术层级	具体技术	在项目中的典型应用场景
后端框架	Spring Boot 2.x, Spring MVC, Spring	快速构建、依赖注入、MVC分层。
数据持久层	MyBatis/MyBatis-Plus, MySQL	实现ORM映射，操作核心业务数据（用户、订单、菜品）。
缓存与会话	Redis	缓存菜品、购物车数据、分布式会话存储、配合实现分布式锁。
前端与移动端	管理端：Vue.js + ElementUI
用户端：微信小程序	管理后台采用组件化开发；用户端提供原生APP体验。
中间件与工具	Nginx（反向代理/负载均衡）、JWT（无状态认证）、Spring Task（定时任务）、WebSocket（实时播报）、阿里云OSS（存储图片）、Apache POI（报表导出）、Git（版本控制）、Maven（项目管理）等。	支撑高并发、实时通信、文件存储等高级需求。
✨ 二、核心亮点与设计精粹
除了技术栈的全面，该项目真正的价值在于对核心业务和典型技术难题的工程化实现。

严谨的业务流程设计

订单状态机：订单从待付款 -> 待接单 -> 已接单 -> 派送中 -> 已完成 的完整状态流转，逻辑严密，是理解业务系统的绝佳模型。

购物车设计：区分了用户端购物车（存于Redis，性能高）和订单快照（下单时持久化到MySQL，保证数据历史性），设计思路清晰。

完整的分层与权限控制

标准MVC分层：Controller -> Service -> Mapper 结构清晰，职责分明。

精细化权限：基于Spring AOP或拦截器实现统一身份校验，并结合自定义注解实现细粒度的功能权限控制（如管理员、店员不同角色）。

典型分布式场景解决方案

数据一致性：在“启售/停售套餐”等场景，通过 Redis分布式锁 防止并发操作导致的数据错乱。

定时任务：使用 Spring Task 或 Quartz 每日凌晨统计前一天的营业额，并生成报表。

消息推送：管理端新订单的语音播报，通过 WebSocket 实现服务端到浏览器的实时消息推送。

性能与安全的工程实践

缓存优化：将高频访问的菜品分类、热门菜品缓存至Redis，大幅减轻数据库压力。

文件存储：菜品图片上传至阿里云OSS，解决本地存储的扩展性和访问效率问题。

安全性：密码MD5加密存储，敏感数据传输加密，接口防刷等基础安全措施。

💡 三、项目定位与学习价值
“苍穹外卖”的本质是一个为教学而生的工业级模拟项目。

维度	分析
项目定位	并非追求极致性能的高并发系统，而是Java技术生态的“集大成”应用演示，覆盖后端开发绝大多数核心知识点。
学习价值	1. 业务理解：深入理解外卖行业的完整业务流程和数据模型。
2. 技术整合：学习如何将十几种技术栈有机整合，解决具体业务问题。
3. 简历亮点：一个功能完整、技术栈主流的项目，是应届生或初级开发者简历上的有力实证。
潜在挑战	作为教学项目，在微服务拆分、数据库深度分库分表、全链路监控等超大规模分布式场景上有所简化，学习者可在此基础上自行探索。
